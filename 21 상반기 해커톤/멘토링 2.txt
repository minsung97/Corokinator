되묻는거 단비 AI
흐름은 기능상 되어야 하니까 그거 먼저 구현
대화 흐름이 AI스럽지 않다고 해도 점수에 큰 영향을 받지 않음
해커톤의 결과물이 얼마나 괜찮은 서비스인지가 더 큰 점수

질문 : 개발일지
캐릭터 만족스러워하심
UI 신경을 잘씀 - 기능만 뒷받침되면 좋을듯
멘트보다는 결과가 정확히 나올것
예, 아니오로 이어져서 코로나인지 아닌지 
너는 이런걸 조심하면 좋아 이런 데이터를 기반으로 추천하는 멘트 추가해주면 더 좋을듯
인공지능 보다는 선별해서 하는건 정해진 데이터의 트리이기 때문에 어쩔수는 없지만,
그 데이터를 기반으로 추천 또는 다른 서비스를 연결하면 좋을 듯

창호님 : 엔티티 사용질문
예, 아니오만 사용하고 싶은 경우에는 중의적 표현을 모르는 것으로 두고 되묻는게 좋을지?
답변 : 자체적으로 답변을 만들어도 상관없음
젊은 사람들이 쓸 만한 내용은 없음
채팅에서 사용하는 내용이 거의 없음
코로나는 신뢰할만한 데이터가 중요하기 때문에 오진단보다는 정확하게 진단하는게 신뢰성을 더 높일 수 있음
창호님 : 슬롯(?) 끼리 연결하는 방식씀
답변 : 훨씬 괜찮음 - 사용자한테 다른 선택지를 주는 것보다 정확하게 입력을 받는게 좋음
진단이 목적이기 때문에 못 벗어나게 하는게 더 좋음
창호님 : 질문화를 더 다양화하는 방법?
답변 : 조건절로 가능, 크거나 작거나, 포함 등 다 가능

주형님 : 웹 사이트에서는 버튼 이벤트해서 단비 호출이 가능한데 반대로 가능한지? 
답변 : 가능함 - 액션 스크립트
parent:함수(); 를 쓰면 웹사이트의 함수를 호출해줌 - 게시판에 다시 물어봐주면 상의 함수를 호출하는 방법을 답변해주겠음
상호 연동이 가능

창호님 : 하나의 슬롯에 여러개 질문 저장 가능?
답변 : 제공 안 함. 추가 예정
창호님 : 비슷한 역할 수행? 질문 답변이 같아도 일정 확률로 다른 슬롯으로 빠지도록?
답변 : 없음 - 펑션 노드에서 하거나, 사용자한테 조건을 주고 넘기는 방식으로 사용

주형님 : knowledge, api 노드에서 엑셀 파일이 한행씩 밖에 안가져와짐
답변 : 아직 여러행 못 가져옴
결론 : api만 사용가능
